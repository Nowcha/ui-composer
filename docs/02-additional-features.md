# 追加機能提案

カタログ統合・キャンバス・3形式出力(設計書v1)に加えて、詰め込む価値のある機能の提案。
優先度は「AIへの指示精度が上がるか」を基準に判定。

## S: 必ず入れたい(AI指示の質に直結)

### 1. 差分プロンプト生成 ★イチオシ
スペックのスナップショットを保存し、前回との差分だけを抽出したプロンプトを生成する。
「Card内にSwitchを1つ追加、ラベル変更」のような最小指示が出せるため、
Claude Codeでの**反復開発が劇的に速くなる**(毎回全体スペックを渡すとAIが無関係な箇所まで書き換えるリスクがある)。

- 実装: SpecTreeのスナップショット履歴(localStorage) + ツリーdiffアルゴリズム
- 出力例: 「以下の差分のみ実装。他のコンポーネントは変更しないこと: ...」

### 2. デザイントークンエディタ
プライマリカラー・角丸・spacing・フォントを一度定義すると、
全出力(プロンプト冒頭の規約ブロック / CLAUDE.mdスニペット / 生成コードのtailwind.config)に自動反映。
「毎回色を指示し忘れて後から直す」問題を根絶する。

### 3. 画面テンプレート(プリセット)
ログイン / 設定 / ダッシュボード / CRUD一覧+詳細 / フォームウィザード の5パターンを同梱。
自作レイアウトのテンプレート保存・呼び出しも可能に。
ゼロから組む頻度を下げ、「テンプレ呼び出し→微調整→差分プロンプト」のループを作る。

## A: 入れたい(体験が大きく向上)

### 4. URL共有(サーバーレス)
SpecTreeをlz-string圧縮してURLハッシュに埋め込む。
GitHub Pagesのままリンク1本でレイアウトを共有できる(スマホで開いて確認、も可能)。

### 5. a11yリント
スペック保存時に簡易チェック: ラベルなしInput、アイコンのみでaria-labelなしのButton、
見出し階層の飛びなど。検出結果はプロンプトに「実装時の注意」として自動注記。
カタログのa11yNotesと連動。

### 6. 画面遷移エディタ(Mermaid出力)
複数画面を作成し、「このButton→あの画面」の遷移を線で結ぶ。
Markdownスペックに遷移図(Mermaid flowchart)を自動挿入。
単画面ツールから「アプリ全体をAIに指示できるツール」に格上げされる。

### 7. ダミーデータ指示の自動生成
TableやListに「行数・カラム構成・日本語ダミーデータの性質(人名/金額/日付)」を設定でき、
プロンプトに「以下のダミーデータで表示確認できる状態にすること」を含める。
AIが空のUIを納品してくる問題を防ぐ。

## B: 余裕があれば

### 8. AIレビューモード(Claude in Claude・任意)
Artifacts内蔵のAPI(キー不要)でスペックJSONをClaudeに渡し、
「この設計の曖昧点・実装者が迷う箇所」を指摘させる。送信前のセルフレビューとして機能。
※ 本体はAPIキー不要原則のまま、この機能だけArtifact側に分離する手もある

### 9. キーボードショートカット + コマンドパレット
Cmd+K でコンポーネント検索→即配置。作業速度に効くが後回し可。

### 10. スペックのバージョンタグ + 履歴ビュー
差分プロンプト(#1)の基盤の上に、名前付きバージョン(v0.1「初版」等)と履歴一覧UI。

## 見送り推奨

- **Figmaインポート/エクスポート**: 工数対効果が悪い。差分プロンプトの方が実利が大きい
- **リアルタイム共同編集**: サーバー必須になり原則違反
- **画像からの逆生成**: API必須。必要ならClaude Code側で直接やればよい
