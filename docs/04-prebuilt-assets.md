# 事前準備資産リスト(コンポーネント・アイコン以外)

> components.json / icons.json に加えて、静的データとして事前に仕込んでおくと出力品質と作業速度が上がる資産の一覧。
> いずれもビルド時静的JSON(またはMarkdown)で、APIキー不要原則を維持。
> 生成は基本Sonnetでバッチ実行可(各項目に生成指示を併記)。

---

## S: Phase 1〜2 で仕込む(効果大・軽量)

### 1. プロンプト規約スニペット集 `src/data/prompt-rules.json`
全量/差分プロンプトの冒頭に自動挿入される共通規約のプリセット。
- 例: 「レスポンシブ対応(mobile-first)」「エラー/ローディング/空状態を必ず実装」「アイコンはPhosphor weight=regular統一」「日本語UIテキスト」「アクセシビリティ: フォーム要素に必ずlabel」
- ユーザーがON/OFFできるチェックボックス形式(15〜20項目)
- **これが「毎回指示し忘れて手戻り」を潰す最重要資産**
- 生成指示: Claude Codeへの指示で頻出する品質要件を、UI実装/レポート生成の2カテゴリで各10項目作成

### 2. 日本語ダミーデータセット `src/data/dummy-data-ja.json`
Table/List/カードに流し込む現実的な日本語データ。AIが空UIを納品する問題と「Lorem ipsum感」を同時に解決。
- 人名(姓名かな付き20件)、会社名(架空10件)、部署/役職、住所(架空)、メール、日付、金額(¥表記)、ステータス語彙(承認待ち/完了/差戻し等)、商品名
- プロンプトには「以下のダミーデータで表示確認可能な状態にすること」として自動添付
- 生成指示: 実在と紛らわしくない架空データであることを制約に明記

### 3. 日本語フォント & タイポグラフィプリセット `src/data/typography-presets.json`
デザイントークンの初期値として選べるペアリング集(5〜6種)。
- 例: Noto Sans JP(標準)/ Zen Kaku Gothic New(モダン)/ BIZ UDPGothic(ビジネス・可読性)/ M PLUS 1p 等
- 各プリセットに Google Fonts の読込タグ、見出し/本文サイズスケール、行間(日本語は英文より広め: 1.7〜1.9)を含める
- レポートモードで特に効く

### 4. カラーパレットプリセット `src/data/color-presets.json`
Tailwind標準パレットをベースに、用途別の組み合わせ(8種程度)。
- 例: コンサル資料(navy+gray+accent)、ダッシュボード(dark)、ヘルスケア(teal)等
- 各パレットに semantic 割当(primary/success/warning/danger/surface)と、WCAG AAコントラスト検証済みの文字色ペアを含める
- 生成指示: コントラスト比を実際に計算し、AA未満の組合せを含めないこと

## A: Phase 3 で仕込む(レポートモードと同時)

### 5. チャート選定ガイド `src/data/chart-guide.json`
チャートプレースホルダのプロパティ選択を支える「データ性質→適切なチャート」対応表。
- 時系列推移→折れ線 / 構成比→積上げ棒or円(5分類以下) / 比較→横棒 / 相関→散布図 / 進捗→ゲージ 等(12種)
- 各項目に「AIへの指示文テンプレ」(ライブラリ指定: Recharts前提の実装注意)を付与
- プロンプト出力時、選んだチャート種の指示文が自動挿入される

### 6. レポート構成テンプレート `src/data/report-structures.json`
HTMLレポートの定番構成(アウトライン)集。
- AI相談結果共有(背景→論点→比較→結論→次アクション)、調査レポート、意思決定資料(エグゼクティブサマリー先頭)、進捗報告 の4種
- レポートモードで選ぶと、Sectionノードのひな形がキャンバスに展開される

### 7. a11yルール集 `src/data/a11y-rules.json`
リント機能の判定ルール定義(コードから分離してデータ化)。
- ラベルなし入力、アイコンのみボタンのaria-label欠落、見出し階層飛び、コントラスト警告、タッチターゲット44px 等(15ルール)
- 各ルールに「検出条件(ノードパターン)」と「プロンプト注記文」をセットで定義

## B: Phase 4 で仕込む

### 8. 画面/レイアウトパターン辞書 `src/data/layout-patterns.json`
テンプレート機能の下敷き。サイドバー型、ダッシュボードグリッド、マスター/ディテール、ウィザード、設定ページ等の骨格(SpecTree断片)10種。

### 9. 逆変換の対応表 `src/data/import-mappings.json`
HTML直接インポート(経路B)の認識精度を上げるマッピング辞書。
- 既知パターン(Tailwindの典型クラス構成、table構造、alertロール等)→カタログコンポーネントの対応ルール
- 自ツール製HTMLレポートの `data-uic-type` はここを参照して復元

## 生成の進め方

- 1〜7は**Sonnetでバッチ生成可能**(各JSONにスキーマ→バリデーション→生成、の順。01-catalog-generation.mdと同じ型)
- Opusが必要なのは #9(マッピング規則の設計)のみ
- 推奨: `prompts/02-assets-generation.md` として catalog と同形式の生成プロンプトにまとめ、`npm run gen:assets` で再生成可能にする
- 追加時のCLAUDE.md追記: 「src/data/*.json はすべて scripts/ 生成物、手編集禁止」ルールを assets にも適用

## 見送り(調査したが不要と判断)

- **UIパターン解説文の収録**(ui-patterns.com等の文章): プロンプトに載せてもAIは既知。カタログのdescriptionで十分
- **画像素材/イラスト集**: ワイヤーフレーム忠実度の方針と矛盾。プレースホルダ矩形で十分
- **英語ダミーデータ**: 用途が日本語UIなので優先度なし。必要になったら追加
