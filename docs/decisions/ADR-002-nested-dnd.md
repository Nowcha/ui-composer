# ADR-002: ネストD&Dの実装方式

- 日付: 2026-07-12(Session 3)
- ステータス: 採用

## コンテキスト

パレット→キャンバスへの新規配置と、ツリー内の並び替え・ネスト移動を
dnd-kit で実現する。MVP最大の難所(docs/03 Session 3)。

## 決定事項

1. **衝突判定は `pointerWithin`**(CLAUDE.md規約どおり)。`closestCenter` は
   ネスト構造で親コンテナと子の中心が近接し誤判定するため使わない
2. **SortableContext を親ごとに入れ子**にし、同一親内の並び替えはsortableの
   標準挙動に任せる
3. **ドロップ先IDのエンコード規則**(src/canvas/dnd-ids.ts):
   - ノード行に重ねる → その親の該当インデックスに挿入(並び替え)
   - `into:{nodeId}` ゾーンに重ねる → コンテナ末尾に追加(ネスト)
   - パレット項目は `palette:{componentId}` で新規ノード生成
4. **「この中にドロップ」ゾーンはドラッグ中のみ表示**(空コンテナは常時)。
   行への重なり=並び替え、ゾーンへの重なり=ネスト、を視覚的に分離する
5. **インデックス補正なしの「削除→挿入」方式**: `moveNode` が先に削除して
   から挿入するため、同一親内の下方向移動でも `arrayMove` と同一結果になる
   (tree-utils.test.ts で担保)
6. **循環防止はストアではなく純関数側**: `moveNode` が自己サブツリーへの
   移動を拒否し入力参照をそのまま返す → ストアは参照同一性で履歴汚染を防ぐ
7. PointerSensor に `distance: 5` の activationConstraint を設定し、
   クリック操作(選択・クリック追加)とドラッグを共存させる
8. KeyboardSensor は導入しない(行のbutton要素とSpaceキーが衝突するため。
   キーボードでの移動操作は Phase 4 のコマンドパレットで別途提供予定)

## 却下した代替案

- **DragOverlayなしの素のtransform**: パレットからのドラッグで元要素が
  動いて戻る挙動が不自然 → ラベルだけのDragOverlayを採用
- **onDragOverでのリアルタイムツリー変更**(dnd-kit公式Tree例の方式):
  実装が複雑でundo履歴が汚れる。ドロップ確定時(onDragEnd)のみ変更する
